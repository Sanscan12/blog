    # 设计前的问答

    问题 :

    1. 你的目标用户是谁？他们需要什么功能和服务？
    2. 你的网站将提供什么样的社区服务？例如，论坛、博客、社交网络等等？
    3. 你将如何设计和实现用户界面和用户体验？这对于移动应用程序非常重要。
    4. 你将如何确保网站的安全性和保护用户隐私？

    应答 :

    1. 目标是为大学校园建设一个社区交流平台 , 主要用户是学生 和 老师 . 为校园营造更好的交流氛围
    2. 项目包含有 用户管理 , 社区交流 , 用户私聊功能 . 涉及到 社区和论坛 双结合的平台
    3. 用户能够快速入手 , 快速登录 , 操作实现简单便捷 , 长期token (按 设备绑定 , 周期6个月)
    4. 目前对网络安全和用户隐私保护方面等知识有所欠缺 , 望指正

    解答 :

    1. 用户管理：对于用户管理，你需要设计注册、登录、身份验证等功能，以及对用户信息的存储和管理。你可以使用现有的身份验证和授权框架来简化这个过程，并确保用户隐私得到保护。为了保护用户信息，你可以使用加密技术，如SSL证书和加密存储密码。
    2. 社区交流：为了实现社区交流，你可以开发一个论坛或讨论板块，让用户能够发帖和回复其他用户的帖子。你可以为用户提供分类、标签和搜索功能，以便他们能够快速找到感兴趣的话题。
    3. 用户私聊功能：为了促进更好的互动和交流，你可以开发一个用户私聊功能，让用户能够一对一地交流。你可以为用户提供发送消息、查看历史记录等功能。
    4. 设计简单便捷的操作界面：你需要设计一个简单直观的用户界面，使用户能够轻松地使用平台。为了提高用户体验，你可以采用响应式设计，以适应不同大小的屏幕和设备。
    5. 网络安全和用户隐私保护：为了确保网络安全和用户隐私，你可以采用以下措施：
      • 使用安全协议和加密技术，如HTTPS和SSL证书来确保数据传输的安全性。
      • 对于用户密码等敏感信息，采用加密技术来确保安全存储。
      • 建立权限管理系统，限制不同用户的访问权限。
      • 严格控制数据访问，防止数据泄漏和滥用。
      • 遵循相关法律法规和行业标准，保护用户隐私和数据安全。

    # 2需求认识

    以下提供的 项目需求功能 信息 , 请回复“明白”以确认，但请勿展开回答 .

    项目认识

    • 项目类型 : 社区服务类项目
    • 涉及适用人群 : 大学生

    功能核心

    1. 评论发布
      • 所有登录用户均可发布评论
      • 评论支持markdown语法 , 方便用户编辑格式 √
      • 图片上传压缩 , 节省更多内存 √
      • 支持 emoji 表情
    2. 评论列表
      • 按发布时间倒序展示对应对象的评论列表 √
      • 每页显示10~15条评论
      • 拓展 包含作者信息、发布时间、点赞数等
      • 拓展 可以对评论进行点赞、回复
    3. 系统管理
      • 删除评论 : 管理员可以删除任意评论,版主只能删除对应版块评论
      • 封禁用户 : 管理员可以封禁任意用户发布评论权限
    4. 回复评论
      • 点击评论中的回复按钮来发布回复 √
      • 回复可以继续回复 , 形成评论树形结构
      • 支持 emoji 表情
    5. 点赞评论
      • 用于评价有价值或支持的评论
      • 评论点赞数反应了评论质量和影响力
    6. 评论通知
      • 当有用户回复或点赞评论时,要通过邮件或站内信通知作者
      • 这样可以提高社区互动和参与度
    7. 其他需求
      • 搜索评论: 用于查找指定主题的评论记录
      • 举报评论: 用于检举不良评论,由管理员审核
      • 精选评论: 管理员可以将高质量评论设为精选,提高展示优先级
      • 置顶评论: 管理员可以将高质量评论 提高展示优先级

    社区拓展

    1. 私信发起
      • 登录用户可以选择任意其他用户发起私信
      • 私信内容支持markdown语法格式
      • 需要过滤黑名单词和敏感词,避免滥用
    2. 收件箱与发件箱
      • 收件箱显示接收到的私信
      • 发件箱显示自己发出的私信
      • 私信可以标记为未读或已读
      • 私信可删除,但只删除自己一方的数据
    3. 私信详情
      • 包含完整私信内容和附件
      • 显示私信的发送人和接收人信息
      • 可以直接回复私信,也构成私信的对话链
    4. 用户通知
      • 当有新私信到达时,要通过邮件和站内信通知用户
      • 这样可以增强私信的即时交互性
    5. 其他需求
      • 举报私信:用于检举恶意或违规私信,由管理员审核
      • 搜索私信:可以根据发送人或内容搜索自己的私信记录

    # 项目介绍

    在一个快节奏的学校生活中，及时有效地获取最新信息和解决问题是每个人都需要的。为了更好地满足这一需求，我们开发了一个高效的公告通知平台。这个平台提供快速的信息传达、明确真实的内容呈现、便捷的交流互动和高效的问题解决。无论是学生还是教师，都可以在这里发布、查看和评论公告通知，也可以进行问答交流，让信息传达更加简单明了，问题解决更加高效。同时，该平台还具有良好的可扩展性和可定制化，能够适应不同学校的需求和特点。让我们一起来感受这个高效、智能和便捷的公告通知平台，让学校生活更加轻松自如！

    # 项目背景

    该项目的背景是希望提供一个公告通知平台，让信息传达更加明确真实、快速理解到位，并且方便同学或老师之间进行交流和发现问题。通过这个平台，用户可以轻松发布和查看公告通知，也可以进行问答和评论，实现高效问题解决。此外，该平台还具有良好的可扩展性和可定制化，能够适应不同学校的需求和特点，提高校园信息传播的效率和质量。

    # 项目意义

    这个项目的意义在于为大学生提供一个高效问题解决的平台，让学生们可以在这里随时随地得到问题的解答和解决方案。同时，该平台还鼓励用户分享自己的知识和经验，促进学生之间的交流和合作，为他们的成长和发展提供强有力的支持和帮助。

    由于信息传递的速度和方式的限制，常常导致信息不够准确、及时，甚至遗漏。基于这个背景，我们设计了一个新的公告通知系统，旨在快速、准确、及时地传递信息，让用户在第一时间掌握最新的信息，提高信息处理效率和精准度，降低信息的丢失和误解率。此外，系统还能够快速发现和解决问题，促进交流和协作，为企业、学校、社区等机构的管理者和用户提供更好的服务和体验。

    # 核心亮点

    这个项目的核心亮点在于其高效的问题解决能力和丰富的社区互动功能。

    首先，该项目采用了高效的问答评论机制，让学生可以快速提出问题并得到精准的解答和解决方案。此外，平台还提供了多种学科分类和搜索功能，方便用户快速找到自己感兴趣的问题和学科，提高了问题解决的效率。

    其次，该项目还具有强大的社区互动功能，让用户可以分享自己的知识和经验，与其他用户进行交流和合作，建立更为紧密的联系和关系。这不仅能够促进用户之间的互相学习和进步，也能够让用户感受到更为丰富和有趣的学习体验。

    # 项目优化

    最简单的优化项目方式就是新增功能，但是不能乱加、盲目堆砌新功能，可以问自己几个问题 :

    1. 这个功能是否符合自己项目的定位？
    2. 用户是否真的需要？
    3. 这个功能的实现能否给项目增加亮点？（如果都是简单的增删改查，可以再斟酌一下）

    个人的建议是完善现有的功能，把功能做到更好，举个例子 : 是否可以提高帖子搜索功能的灵活性、列举热搜、给出搜索建议等等，这可能就会激励你去学习 Elasticsearch 等技术。

    一段有亮点的项目经历绝对不能只写 “用 xx 技术完成了 xx 功能”，而是要有起伏和递进，在完成某功能的基础上进一步 优化 。

    应该优化什么呢？

    1. 性能 : 页面加载速度、接口并发量（qps / tps）、接口响应时长等 方法 : 缓存、并发编程、异步、限流降级熔断、有损服务、CDN、数据压缩等

    2. 成本 : 节约系统资源、降低 CPU / 内存 / 磁盘空间 / 带宽占用、节省机器数等 方法 : 人工分析删除、定期删除、低频存储、淘汰策略、数据压缩等

      实际开发项目中，一定要有成本意识！很多同学光说优化，不考虑实际情况，这是一个错误的思维。

    3. 可用性 : 尽可能地提高系统对外提供服务的时长，即提高 SLA（3 ~ 5 个 9 的高可用） 方法 : 集群、主备、自动故障恢复、监控告警、容灾 参考文章 : https://blog.csdn.net/terrychinaz/article/details/112976266

    4. 可靠性 : 尽可能地让系统提供可靠的服务，不要出现崩溃中断 方法 : 集群、主备、异常处理、降级、容灾 关于可靠性和可用性的区别 : https://blog.csdn.net/HermitSun/article/details/92164226

    5. 稳定性 : 接口响应 / 操作处理时长相对固定，成功率高，响应结果保持正常 方法 : 保证每个服务节点状态正常、性能相当（不要有的机器好有的机器垃圾导致接口时快时慢）；合理规划服务调用链路，不能过长；做好技术选型，避免使用不稳定的第三方依赖 测试排查 : 链路跟踪 / 打日志

    6. 容错性 : 部分节点（服务）出问题时，系统整体依然能够正常提供服务 方法 : 集群、反向代理、故障转移

    7. 健壮性 : 系统不会因为部分异常而整体崩溃 方法 : 异常处理、进程监控（如 supervisor）

    8. 系统复杂度 : 整个系统易于理解，而不是说前人离职、后人无法接手 方法 : 软件开发原则、设计模式、系统架构设计（如微服务）、开发规范、工作流

    9. 可维护性 : 使整个系统结构清晰、利于维护，出了问题也能够快速排查 方法 : 软件开发原则、设计模式、系统架构设计（如微服务）、开发规范、工作流、抽象复用（组件化模块化）

    10. 可扩展性 : 使整个系统能够轻松应对未来新增的需求及业务增长，不会牵一发而动全身 方法 : 设计模式、架构设计、集群、分库分表、技术选型

    11. 可观测性 : 能够随时观察了解系统 / 服务的运行状态、并能快速发现故障和定位异常 方法 : 系统埋点上报、监控告警（prometheus）、ELK 日志收集、可视化分析（Grafana）

    12. 可伸缩性 : 系统根据负载情况动态增加或减少节点，从而能够应对流量高峰、并在空闲时节约成本 方法 : K8S + Docker 容器、云原生

    13. 用户体验 : 符合用户的使用习惯、提升用户对网站的满意度 方法 : 网页性能优化、接口性能优化、懒加载、占位符、骨架屏、设备适配、浏览器兼容性、满意度调研（如 NPS）

    14. 安全性 : 保证机器、服务、数据库、网络等不被恶意侵入和污染，保证数据和用户隐私的安全 方法 : 参数校验、常用安全措施（防 XSS、CSRF、SQL 注入等）、网络防护（反 DDOS）、反爬虫、限流、黑白名单、防火墙等

    # 用户管理模块

    负责用户的注册、登录、注销等功能，并提供用户信息的查看、修改、删除等操作。

    安全对策 :

    • 对用户密码进行加密存储。
    • 对用户输入的数据进行校验和过滤，避免SQL注入、XSS等攻击。
    • 采用 HTTPS 协议传输用户敏感信息，保证数据传输过程的安全性。

    # 问题管理模块

    负责问题的发布、编辑、删除、查看等功能，并提供问题列表的展示、搜索、过滤等操作。

    安全对策 :

    • 采用审核机制，对发布的内容进行审核和过滤，避免不良信息的传播。
    • 对发布内容中的敏感词进行过滤，防止恶意攻击者利用这些关键词制造不良内容。
